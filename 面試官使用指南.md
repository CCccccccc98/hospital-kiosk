# 醫院報到系統 - 面試官使用指南

## 🚀 快速啟動

### 方式一：執行可攜式應用程式（推薦）

1. 解壓縮應用程式資料夾
2. 進入 `dist-electron\win-unpacked` 資料夾
3. 雙擊 `醫院掛號系統.exe`
4. 應用程式會自動啟動，包含前端介面和後端伺服器


## 📋 系統功能展示

### 1. 患者報到流程

**測試用身分證字號**（已在資料庫中）：
- `A123456789` - 陳小美
- `B234567890` - 林志豪
- `C345678901` - 張雅婷
- `D456789012` - 王大明
- `E567890123` - 李國華

**操作步驟**：
1. 點擊首頁的「患者報到」
2. 輸入身分證字號（例如：A123456789）
3. 系統會自動驗證並顯示患者姓名
4. 選擇要掛號的診間
5. 點擊「確認報到」
6. 取得號碼牌（會顯示號碼和診間資訊）

**展示重點**：
- ✅ 身分證格式驗證（台灣身分證檢查碼演算法）
- ✅ 重複報到檢查（同一患者不能重複報到同一診間）
- ✅ 診間人數上限（每診間最多 10 人候診）

### 2. 醫師叫號功能

**操作步驟**：
1. 點擊首頁的「醫師控制台」
2. 選擇診間（例如：內科一診）
3. 點擊「叫號」按鈕
4. 當前號碼會遞增，候診人數會減少

**展示重點**：
- ✅ 即時更新候診狀態
- ✅ 號碼自動遞增
- ✅ 候診人數自動計算

### 3. 即時看診進度

**操作步驟**：
1. 點擊首頁的「即時看診進度」
2. 查看所有診間的當前號碼和候診人數
3. 資料會即時更新

**展示重點**：
- ✅ 所有診間資訊一覽
- ✅ 清楚顯示當前號碼和候診人數

## 🎯 完整展示流程建議

### 情境：患者報到並叫號

1. **開啟應用程式**
   - 雙擊執行檔啟動系統

2. **查看初始狀態**
   - 進入「即時看診進度」
   - 記下內科一診的當前號碼（例如：12）

3. **患者報到**
   - 返回首頁，點擊「患者報到」
   - 輸入 `A123456789`
   - 選擇「內科一診」
   - 確認報到
   - 取得號碼牌（例如：16 號）

4. **驗證重複報到保護**
   - 再次嘗試用 `A123456789` 報到內科一診
   - 系統會顯示「您已經報到過此診間了！」

5. **醫師叫號**
   - 進入「醫師控制台」
   - 選擇「內科一診」
   - 連續點擊「叫號」數次
   - 觀察號碼遞增（13 → 14 → 15 → 16）

6. **確認資料更新**
   - 返回「即時看診進度」
   - 確認內科一診的號碼已更新

## 🔒 系統特色

### 後端技術
- **Express.js** REST API
- **SQLite** 資料庫（資料持久化）
- **身分證驗證**：完整的台灣身分證檢查碼演算法
- **操作日誌**：所有報到和叫號操作都有記錄

### 安全性功能
- ✅ 身分證格式驗證
- ✅ 重複報到檢查
- ✅ 診間人數上限控制
- ✅ API 錯誤處理
- ✅ React 錯誤邊界

### 資料庫結構
- **patients**：患者資料
- **clinics**：診間資訊
- **checkin_records**：報到記錄
- **operation_logs**：操作日誌

## ❓ 常見問題

### Q: 應用程式無法啟動？
A: 請確認：
- Windows Defender 或防毒軟體沒有阻擋
- 檔案完整解壓縮（不要在壓縮檔內直接執行）

### Q: 資料會保存嗎？
A: 是的，所有報到記錄都儲存在 SQLite 資料庫中，關閉應用程式後重新開啟資料仍在。

### Q: 可以重置資料嗎？
A: 刪除 `server/hospital.db` 檔案，重新啟動應用程式會自動建立新的資料庫。

### Q: 如何重新打包應用程式？
A: 執行以下指令：
```bash
npm run electron:build
```
打包完成的執行檔會在 `dist-electron/win-unpacked` 資料夾中。
